🎯 Workshop 挑戰：2 個核心功能模組

模組 1：異常偵測引擎 🚨

功能說明：針對現有交易資料進行異常模式偵測

  • ✅ 高額交易偵測：單筆金額 > 50,000 元
  • ✅ 頻繁交易偵測：1 小時內同一卡片 > 5 筆交易
  • ✅ 重複交易偵測：5 分鐘內相同卡片、商店、金額

模組 2：警示管理系統 📊

功能說明：管理偵測到的異常交易警示

  • 🔍 警示查詢：依狀態、風險等級、時間範圍篩選
  • 📈 統計報表：警示數量、風險分布、處理狀態
  • ✅ 警示處理：標記為已處理/誤報

💡 偵測機制說明：

方案 A：手動偵測（簡化版）

  • ✅ 透過 POST /api/alerts/detect 手動觸發
  • ✅ 分析 data.sql 中的現有交易資料
  • ✅ 適合 Workshop 快速展示
  • ⚠️ 如需測試不同場景，可修改 data.sql 新增測試資料

方案 B：自動偵測（完整版）⭐ 推薦

  • ✅ 實作 POST /api/transactions 新增交易 API
  • ✅ 每次新增交易時自動觸發異常偵測
  • ✅ 更貼近真實業務場景
  • ✅ 可透過 API 動態新增測試資料
  • 💡 在 TransactionService.createTransaction() 中呼叫偵測邏輯

建議實作順序：

  1. 先實作方案 A（手動偵測）驗證偵測邏輯
  2. 再擴充方案 B（自動偵測）完善系統功能
