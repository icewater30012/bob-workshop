💡 開發流程建議

建議學員採用以下開發流程，充分運用 Bob 的 Plan 與 Code 模式：

1️⃣ 規劃階段 (Plan 模式)

  • 使用 Plan 模式分析需求與設計架構
  • 讓 Bob 協助規劃 Entity、Repository、Service、Controller 的設計
  • 確認資料庫結構與 API 端點設計
  • 產出開發計畫與實作步驟
  • 可請 Bob 產出系統架構圖、資料流程圖、ER 圖等視覺化文件

2️⃣ 開發階段 (Code 模式)

  • 切換至 Code 模式進行實際開發
  • 依照規劃逐步實作各層級元件
  • 使用 Bob 的程式碼生成與重構功能
  • 執行測試並驗證功能

3️⃣ 模式切換技巧

  • 遇到設計問題時，切回 Plan 模式重新規劃
  • 需要程式碼實作時，使用 Code 模式
  • 善用兩種模式的優勢，提升開發效率

💡 提示：Plan 模式適合思考與設計，Code 模式適合實作與除錯。靈活切換可以讓開發更有條理！

🎯 挑戰概述

你的任務

你是資深開發工程師，公司需要為現有的信用卡交易系統新增即時交易監控功能，以識別異常交易模式並提供風險預警。

你將使用 IBM Bob 作為 AI 開發助手，從現有的 Sample Code 基礎上，快速實作這個關鍵的安全功能。

📋 功能需求

你需要實作以下 2 個核心功能模組：

🚨 模組 1：異常偵測引擎

功能說明：針對現有交易資料進行異常模式偵測

  • ✅ 高額交易偵測：單筆金額 > 50,000 元
  • ✅ 頻繁交易偵測：1 小時內同一卡片 > 5 筆交易
  • ✅ 重複交易偵測：5 分鐘內相同卡片、商店、金額

實作方式：

  • 使用現有 TransactionService 查詢方法
  • 分析交易資料（data.sql 或新增的交易）
  • 產生 TransactionAlert 警示記錄

📊 模組 2：警示管理系統

功能說明：管理偵測到的異常交易警示

  • 🔍 警示查詢：依狀態、風險等級、時間範圍篩選
  • 📈 統計報表：警示數量、風險分布、處理狀態
  • ✅ 警示處理：標記為已處理/誤報

實作方式：

  • 建立 AlertRepository 和 AlertService
  • 實作 RESTful API 端點
  • 提供查詢、統計、處理功能

✅ 現有 API 功能

Sample Code 已提供以下交易查詢 API：

HTTP 方法 端點 功能說明
GET /api/transactions 查詢所有交易
GET /api/transactions/{id} 查詢單筆交易
GET /api/transactions/card/{cardId} 查詢指定卡片的交易
GET /api/transactions/recent?hours=24 查詢最近 N 小時交易
GET /api/transactions/high-amount?threshold=50000 查詢高額交易
GET /api/transactions/statistics 交易統計資料

💡 實作方案選擇

方案 A：手動偵測（簡化版）

  • 透過 POST /api/alerts/detect 手動觸發偵測
  • 分析 data.sql 中的現有交易資料
  • 適合 Workshop 快速展示

方案 B：自動偵測（完整版）⭐ 推薦

  • 實作 POST /api/transactions 新增交易 API
  • 每次新增交易時自動觸發異常偵測
  • 更貼近真實業務場景
  • 可透過 API 動態新增測試資料

建議：先實作方案 A 驗證偵測邏輯，再擴充方案 B 完善功能

📖 如何使用 Sample Application 完成挑戰

本專案提供完整的 Sample Application，包含基礎交易查詢功能。你需要在此基礎上新增警示監控功能。

步驟 1：Clone 專案

git clone https://github.com/owhc/bob-workshop.git
cd bob-workshop/topics/transaction-monitor

步驟 2：啟動應用程式

選擇以下任一方式：

方式 A：使用 Maven Wrapper（推薦）

./mvnw spring-boot:run

方式 B：使用已安裝的 Maven

mvn spring-boot:run

步驟 3：驗證啟動成功

看到以下訊息表示啟動成功：

Started TransactionMonitorApplication in X.XXX seconds

步驟 4：探索現有功能

  • Web UI: http://localhost:8080 - 查看交易列表
  • H2 資料庫控制台: http://localhost:8080/h2-console - 檢視資料庫內容
    • JDBC URL: jdbc:h2:mem:uitc_db
    • Username: sa
    • Password: (留空)
  • API 文件: http://localhost:8080/swagger-ui.html - 測試現有 API

步驟 5：使用 Bob 開始開發

在 IBM Bob 中開啟專案，開始實作警示功能：

  • 📖 閱讀 README.md 了解專案結構
  • 🔍 使用 Bob 分析現有的 Transaction.java 和 TransactionService.java
  • 💡 請 Bob 協助建立 TransactionAlert Entity
  • ⚙️ 實作異常偵測邏輯
  • 🔌 建立 Alert 相關 API 端點

⚠️ 常見問題

Q: Port 8080 已被佔用？

A: 修改 src/main/resources/application.yml 中的 server.port

Q: Maven 下載依賴很慢？

A: 設定 Maven 使用國內鏡像源

Q: 如何停止應用程式？

A: 在終端機按 Ctrl + C
