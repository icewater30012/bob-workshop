2. IBM Bob IDE（必要）

⚠️ 請在 Workshop 前 3 天完成安裝

▶ 🍎 macOS 安裝步驟：

  1. 下載安裝檔：https://bob.ibm.com/download
  2. 開啟下載的 .pkg 檔案
  3. 依照安裝精靈步驟：
    • 同意授權條款
    • 選擇安裝位置（建議使用預設）
    • 輸入管理員密碼
    • 完成安裝
  4. 從應用程式資料夾啟動 Bob

▶ 🪟 Windows 安裝步驟：

  1. 下載安裝檔：https://bob.ibm.com/download
  2. 執行下載的 .exe 安裝程式
  3. 依照安裝精靈步驟：
    • 同意授權條款
    • 選擇安裝目錄（建議使用預設）
    • 選擇開始功能表資料夾
    • 點擊「完成」
  4. 從桌面捷徑或開始功能表啟動 Bob

▶ 🐧 Linux 安裝步驟：

  1. 下載對應的安裝包（.deb 或 .rpm）
  2. 使用套件管理器安裝： 
    # Debian/Ubuntu
sudo dpkg -i bob-installer.deb

# Red Hat/Fedora
sudo rpm -i bob-installer.rpm
  3. 從應用程式選單啟動 Bob

4. Java 開發環境配置（Spring Boot 專案必要）

針對 Workshop 中的 Spring Boot 專案開發，所有參加者都需要安裝 JDK（專案已內建 Maven Wrapper，無需另外安裝 Maven）

步驟 1：Bob IDE 使用者 - 安裝 Java Extensions

如果您使用 Bob IDE，請先安裝以下 Extensions：

核心 Java 支援
  • Extension Pack for Java
    • 包含：Language Support for Java, Debugger for Java, Test Runner for Java, Maven for Java, Project Manager for Java, Visual Studio IntelliCode
    • 提供完整的 Java 開發環境
Spring Boot 開發
  • Spring Boot Extension Pack
    • 包含：Spring Boot Tools, Spring Initializr Java Support, Spring Boot Dashboard
    • 簡化 Spring Boot 應用程式開發與管理

安裝步驟：

  1. 開啟 Bob IDE
  2. 點擊左側活動列的「Extensions」圖示（或按 Ctrl+Shift+X / Cmd+Shift+X）
  3. 在搜尋框中輸入 Extension 名稱
  4. 點擊「Install」按鈕安裝
  5. 安裝完成後，重新載入 Bob IDE（如需要）
💡 提示：Extension Pack 會自動安裝所有相關的 Extensions，建議優先安裝 Pack 版本以節省時間。專案已使用 H2 Console 進行資料庫管理，無需額外安裝資料庫工具。

步驟 2：安裝 JDK（Java Development Kit）

Workshop 專案使用 Spring Boot，建議安裝 JDK 17（可向下相容編譯 Java 11 程式）。

▶ 🍎 macOS 安裝 JDK：

  • 方式 A：使用 Homebrew（推薦） 
    # 安裝 OpenJDK 17
brew install openjdk@17

# 設定環境變數
echo 'export PATH="/opt/homebrew/opt/openjdk@17/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
  • 方式 B：使用官方安裝程式
    1. 前往 https://adoptium.net/
    2. 下載 Temurin JDK 17（LTS）for macOS
    3. 開啟 .pkg 檔案並依照指示安裝

▶ 🪟 Windows 安裝 JDK：

  1. 前往 https://adoptium.net/
  2. 下載 Temurin JDK 17（LTS）for Windows
  3. 執行 .msi 安裝程式
  4. 安裝時勾選「Set JAVA_HOME variable」和「Add to PATH」
  5. 完成安裝後重新啟動命令提示字元

▶ 🐧 Linux 安裝 JDK：

  • Ubuntu/Debian 
    # 安裝 OpenJDK 17
sudo apt update
sudo apt install openjdk-17-jdk -y
  • Red Hat/Fedora 
    # 安裝 OpenJDK 17
sudo yum install java-17-openjdk-devel -y

驗證 JDK 安裝：

java -version
javac -version

預期輸出：openjdk version "17.x.x"

💡 關於 Maven：專案已內建 Maven Wrapper (mvnw)，無需另外安裝 Maven。Maven Wrapper 會自動下載並使用正確的 Maven 版本，確保團隊開發環境一致。

5. Eclipse 用戶：Bob Shell

如果您習慣使用 Eclipse IDE，可以透過 Bob Shell 在終端機中使用 Bob 的 AI 功能

📖 什麼是 Bob Shell？

Bob Shell 是一個命令列工具，讓您可以在任何終端機環境中與 Bob AI 互動，無需使用 Bob IDE。

🔧 安裝 Bob Shell

系統需求

  • Node.js 22.15.0 或更新版本
  • 4 GB RAM（建議 8 GB）
  • 500 MB 可用磁碟空間
  • 穩定的網路連線

步驟 1：安裝 Node.js

Bob Shell 需要 Node.js 22.15.0 或更新版本。請依照您的作業系統選擇安裝方式：

▶ 🍎 macOS 安裝 Node.js：

  • 方式 A：使用官方安裝程式（推薦）
    1. 前往 Node.js 官網：https://nodejs.org
    2. 下載 LTS 版本（Long Term Support）
    3. 開啟 .pkg 檔案並依照指示安裝
  • 方式 B：使用 Homebrew 
    brew install node

▶ 🪟 Windows 安裝 Node.js：

  • 使用官方安裝程式
    1. 前往 Node.js 官網：https://nodejs.org
    2. 下載 LTS 版本（Long Term Support）
    3. 執行 .msi 安裝程式
    4. 依照安裝精靈完成安裝（建議勾選「Add to PATH」）

▶ 🐧 Linux 安裝 Node.js：

  • Ubuntu/Debian 
    # 安裝 Node.js 22.x
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
  • Red Hat/Fedora 
    # 安裝 Node.js 22.x
curl -fsSL https://rpm.nodesource.com/setup_22.x | sudo bash -
sudo yum install -y nodejs

驗證 Node.js 安裝：

node --version
npm --version

預期輸出：v22.x.x 或更新版本

步驟 2：安裝 Bob Shell

Bob Shell 使用者請參考第 4 點完成 JDK 的安裝（Maven 已內建在專案中）。

▶ macOS： 

curl -fsSL https://bob.ibm.com/download/bobshell.sh | bash

▶ Windows： 

powershell -ep Bypass 'irm -Uri "https://bob.ibm.com/download/bobshell.ps1" | iex'

▶ Linux： 

curl -fsSL https://bob.ibm.com/download/bobshell.sh | bash

🚀 基本操作

1. 啟動互動式會話

bob

首次使用時會提示您使用瀏覽器登入 IBMid 進行認證

2. 切換模式

Bob Shell 支援多種模式，使用 /mode 指令切換：

  • /mode ask - 詢問模式（解釋概念、回答問題）
  • /mode plan - 規劃模式（設計架構、制定計劃）
  • /mode code - 編碼模式（撰寫和修改程式碼）

3. 與 Bob 互動

範例對話：

# 切換到規劃模式
/mode plan

# 請 Bob 協助規劃
請協助我設計一個 Spring Boot 的交易監控系統架構

# 切換到編碼模式
/mode code

# 請 Bob 生成程式碼
請幫我建立 Transaction Entity 類別

4. 常用指令

  • /help - 顯示說明
  • /mode - 切換模式
  • /clear - 清除對話歷史
  • /quit - 退出 Bob Shell

💡 Eclipse 整合建議

  • 在 Eclipse 中開啟內建終端機（Window → Show View → Terminal）
  • 在終端機中執行 bob 啟動 Bob Shell
  • 使用 Bob Shell 進行程式碼規劃和生成
  • 將生成的程式碼複製到 Eclipse 中進行編輯
⚠️ 注意：Bob Shell 主要用於命令列互動，無法直接整合到 Eclipse IDE 中。建議搭配使用以獲得最佳體驗。

🌐 網路需求

IBM Bob 服務

  • bob.ibm.com - Bob 主要服務

如果您的公司有防火牆限制，請聯絡 IT 部門開放以下連線：

🔧 疑難排解

Q1: Bob 無法啟動

症狀: 點擊 Bob 圖示後沒有反應

解決方案:

  1. 檢查系統需求是否符合
  2. 重新啟動電腦
  3. 重新安裝 Bob
  4. 檢查防毒軟體是否阻擋

Q2: 無法登入 IBMid

症狀: 認證頁面無法開啟或登入失敗

解決方案:

  1. 檢查網路連線
  2. 清除瀏覽器快取
  3. 嘗試使用不同瀏覽器
  4. 重設 IBMid 密碼
  5. 聯絡 IBM 支援

Q3: 網路連線問題

症狀: Bob 無法連接服務或下載失敗

解決方案:

  1. 檢查網路連線
  2. 嘗試使用手機熱點
  3. 聯絡 IT 部門檢查防火牆
  4. 使用 VPN（如果公司有提供）

Q4: 記憶體不足

症狀: 系統變慢或應用程式當機

解決方案:

  1. 關閉不必要的應用程式
  2. 重新啟動電腦
  3. 增加虛擬記憶體
  4. 考慮升級硬體
